# GCC ASM List Files

VS Code extension for GCC/objdump assembly listing files (`.lst`).

Supports RISC-V (RV32IMAC + Zicsr) and Cortex-M33/Thumb-2 targets generated by:

```
objdump -d -S firmware.elf > firmware.lst
objdump -d -S -l firmware.elf > firmware.lst
```

## Features

- **Syntax highlighting** — addresses, hex bytes, mnemonics, registers, hex/decimal literals, comments
- **Hover documentation** — hover over any instruction mnemonic to see its description
- **Go-to-definition** — press F12 on an assembly line to jump to the original C source (requires `objdump -S -l`)

## Architecture auto-detection

Opening a `.lst` file automatically selects the correct grammar:
- `elf32-littleriscv` header → `gcc-lst-riscv`
- `elf32-littlearm` header → `gcc-lst-cm33`

## Path mappings

When the listing was generated inside a Docker container or build VM, the source paths in the file won't exist on your local machine. Configure `gccLst.pathMappings` to rewrite them:

```json
// .vscode/settings.json
{
  "gccLst.pathMappings": [
    { "from": "/build/project", "to": "/Users/me/project" },
    { "from": "/home/ci/src",   "to": "/Users/me/src" }
  ]
}
```

Mappings are tried in order; the first one that produces a file that exists on disk is used. If no mapping matches (or no mapped file exists), the raw path from the listing is used as-is.
